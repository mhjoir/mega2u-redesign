# 🔧 حل مشكلة 404 على Hostinger عند تحديث صفحات mega2u.com

## 🎯 المشكلة
عند تحديث صفحة منتج (مثل `/product/win-11-pro`) أو فتح رابط مباشر، تظهر صفحة **"This Page Does Not Exist"** الافتراضية من Hostinger.

## 🧠 السبب
موقعك تطبيق **SPA (Single Page Application)** مبني بـ React + Wouter. كل المسارات يديرها JavaScript في المتصفح، بينما خادم Hostinger يبحث عن ملفات فعلية بنفس اسم المسار ولا يجدها → يعرض 404.

## ✅ الحل
يكفي رفع ملف `.htaccess` إلى مجلد الجذر `public_html` ليُعيد توجيه جميع الطلبات إلى `index.html`، وعندها يتولى React Router التنقل.

---

## 📋 خطوات التطبيق على Hostinger

### الطريقة 1: عبر File Manager (الأسهل)

1. سجّل دخول إلى **hPanel** على Hostinger.
2. افتح **File Manager**.
3. اذهب إلى مجلد **`public_html`** (حيث يوجد `index.html` الخاص بموقعك).
4. اضغط على زر **Upload** (أو السهم لأعلى).
5. ارفع الملفات الثلاثة:
   - `.htaccess` ← **الأهم**
   - `404.html` ← احتياطي
   - `web.config` ← اختياري (فقط إذا كانت الاستضافة Windows)
6. **مهم:** تأكّد أن ملف `.htaccess` يبدأ بنقطة `.` ولا يتحوّل إلى `htaccess.txt`.
7. حدّث صفحة موقعك بـ **Ctrl+F5** (تحديث قسري).

### الطريقة 2: عبر FTP (FileZilla)

1. اتصل بحسابك عبر FTP.
2. انتقل إلى مجلد `public_html`.
3. اسحب وأفلت الملفات الثلاثة في المجلد.
4. تأكّد من ضبط **"Force showing hidden files"** لرؤية `.htaccess`.

### الطريقة 3: عبر SSH (للمتقدمين)
```bash
cd public_html
nano .htaccess
# الصق محتوى الملف، ثم Ctrl+O للحفظ، Ctrl+X للخروج
```

## 🧪 اختبار الحل

بعد رفع الملف، جرّب الروابط التالية مباشرةً (افتحها في تبويب جديد):

- `https://mega2u.com/product/win-11-pro`
- `https://mega2u.com/cart`
- `https://mega2u.com/login`
- `https://mega2u.com/customer-portal`

ثم اضغط **F5** لتحديث الصفحة. **يجب** أن تظهر الصفحة الصحيحة بدلاً من 404.

## ⚠️ ملاحظات مهمة

1. **عرض الملفات المخفية:** ملف `.htaccess` يبدأ بنقطة، لذا قد يكون مخفياً. في File Manager اضغط على **Settings → Show hidden files**.

2. **إذا لم يعمل الحل:** تحقق من أن `mod_rewrite` مفعّل في Hostinger (عادة مفعّل افتراضياً). راسل دعم Hostinger إذا لزم.

3. **التخزين المؤقت:** قد يحتاج المتصفح لبعض الوقت لتحديث الكاش. استخدم **وضع التصفح الخاص** للاختبار.

4. **API Backend:** إذا أضفت لاحقاً API على نفس الدومين تحت `/api/`، الملف يستثنيها تلقائياً ولن يعيد توجيهها.

5. **HTTPS:** في نهاية ملف `.htaccess` يوجد قسم لإجبار HTTPS — معطّل افتراضياً. فعّله بإزالة `#` بعد التأكّد من تفعيل شهادة SSL.
